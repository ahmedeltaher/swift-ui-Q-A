What is the difference between List and ForEach?

Answer in English

List:

List is a container view that displays a scrollable collection of data in a table-like format. It provides built-in functionality like:

  • Automatic scrolling
  • Row separators
  • Selection handling
  • Swipe actions
  • Pull-to-refresh
List {
    Text("Item 1")
    Text("Item 2")
    Text("Item 3")
}

// Or with data
List(items, id: \.self) { item in
    Text(item.name)
}

ForEach:

ForEach is a view that creates views dynamically based on a collection of data. It's more flexible and can be used inside any container:

ForEach(items, id: \.self) { item in
    Text(item.name)
}

// Can be used in different containers
VStack {
    ForEach(items, id: \.self) { item in
        HStack {
            Image(item.icon)
            Text(item.name)
        }
    }
}

Key Differences:

Aspect List ForEach
Purpose Complete scrollable list container Dynamic view generation
Scrolling Built-in scrolling No built-in scrolling
Styling Table-like appearance with separators No default styling
Flexibility Limited to list format Can be used in any layout
Features Selection, swipe actions, refresh Just view generation

When to Use Which:

Use List when:

  • You want a traditional scrollable list
  • You need built-in list features (selection, swipe actions)
  • You want the standard iOS list appearance

Use ForEach when:

  • You want custom layouts (grids, stacks)
  • You need dynamic views in non-list containers
  • You want full control over styling and behavior

Combined Usage:

List {
    Section("Header") {
        ForEach(items) { item in
            CustomRowView(item: item)
        }
    }
}

الشرح بالعربي

List (القائمة):

دا container جاهز عشان يعرض البيانات في شكل قائمة قابلة للتمرير، وفيه مميزات جاهزة زي:

  • التمرير التلقائي
  • الخطوط الفاصلة بين الصفوف
  • الاختيار
  • السحب للإجراءات (Swipe Actions)
  • السحب للتحديث

ForEach (التكرار):

دا أداة لإنشاء views من مجموعة بيانات، بس مش container بذاته. ممكن تستخدمه جوا أي container تاني.

الفرق الأساسي:

  • List: حل كامل للقوائم مع كل المميزات
  • ForEach: أداة لتوليد views بس، محتاج container

متى تستخدم إيه:

استخدم List لما:

  • عايز قائمة تقليدية زي قوائم iOS
  • محتاج مميزات زي الاختيار والسحب
  • عايز الشكل الجاهز للقوائم

استخدم ForEach لما:

  • عايز layouts مخصصة (Grid، Stack)
  • محتاج views ديناميكية في containers مختلفة
  • عايز تحكم كامل في الشكل والسلوك

مثال عملي:

// List للقوائم التقليدية
List(contacts) { contact in
    ContactRow(contact: contact)
}

// ForEach للـ layouts المخصصة  
LazyVGrid(columns: columns) {
    ForEach(photos) { photo in
        PhotoCard(photo: photo)
    }
}
