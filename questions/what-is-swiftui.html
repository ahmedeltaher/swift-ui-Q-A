What is SwiftUI and how is it different from UIKit?

Answer in English

SwiftUI is Apple's modern declarative UI framework introduced in 2019 that allows developers to build user interfaces across all Apple platforms using Swift. It represents a significant shift from the imperative programming model used in UIKit.

Key differences between SwiftUI and UIKit:

  1. Declarative vs. Imperative: SwiftUI uses a declarative approach where you describe what your UI should look like and how it should behave, and the framework takes care of how to render it. UIKit uses an imperative approach where you manually construct and manipulate views.
  2. State Management: SwiftUI has built-in state management through property wrappers like @State, @Binding, @ObservedObject, and @EnvironmentObject. UIKit requires manual state tracking and UI updates.
  3. Cross-Platform: SwiftUI provides a unified framework for building UIs across all Apple platforms (iOS, macOS, watchOS, tvOS) with minimal platform-specific code. UIKit is primarily for iOS and tvOS.
  4. Live Preview: SwiftUI offers real-time previews in Xcode with Canvas, allowing faster iteration without building and running the app each time.
  5. Automatic UI Updates: When state changes in SwiftUI, the framework automatically updates the UI. In UIKit, you must manually update views in response to data changes.
  6. Layout System: SwiftUI uses a modern layout system with stacks (VStack, HStack, ZStack) and alignments. UIKit uses Auto Layout with constraints.
  7. Composition: SwiftUI has excellent support for creating reusable, composable UI components.
// SwiftUI example
struct ContentView: View {
    @State private var isToggled = false
    
    var body: some View {
        VStack {
            Text("Hello, SwiftUI!")
                .font(.largeTitle)
                .foregroundColor(isToggled ? .green : .blue)
            
            Toggle("Toggle me", isOn: $isToggled)
                .padding()
        }
        .padding()
    }
}
// UIKit equivalent would be much longer with manual setup
class ViewController: UIViewController {
    let titleLabel = UILabel()
    let toggle = UISwitch()
    let toggleLabel = UILabel()
    
    override func viewDidLoad() {
        super.viewDidLoad()
        setupViews()
    }
    
    private func setupViews() {
        // Configure label
        titleLabel.text = "Hello, UIKit!"
        titleLabel.font = .systemFont(ofSize: 34, weight: .bold)
        titleLabel.textColor = .blue
        titleLabel.translatesAutoresizingMaskIntoConstraints = false
        
        // Configure toggle and label
        toggleLabel.text = "Toggle me"
        toggleLabel.translatesAutoresizingMaskIntoConstraints = false
        toggle.addTarget(self, action: #selector(toggleChanged), for: .valueChanged)
        toggle.translatesAutoresizingMaskIntoConstraints = false
        
        // Add to view hierarchy
        view.addSubview(titleLabel)
        view.addSubview(toggleLabel)
        view.addSubview(toggle)
        
        // Set up constraints
        // ...
    }
    
    @objc private func toggleChanged() {
        titleLabel.textColor = toggle.isOn ? .green : .blue
    }
}

الشرح بالعربي

SwiftUI هي إطار عمل حديث من آبل تم تقديمه في عام 2019 لبناء واجهات المستخدم على جميع منصات آبل باستخدام لغة Swift. وهو يمثل تحولًا كبيرًا عن نموذج البرمجة الإجرائية المستخدم في UIKit.

الاختلافات الرئيسية بين SwiftUI و UIKit:

  1. التصريحية مقابل الإجرائية: يستخدم SwiftUI نهجًا تصريحيًا حيث تصف ما يجب أن تبدو عليه واجهة المستخدم وكيف يجب أن تتصرف، ويتولى الإطار مسؤولية كيفية عرضها. بينما يستخدم UIKit نهجًا إجرائيًا حيث تقوم يدويًا ببناء ومعالجة العناصر المرئية.
  2. إدارة الحالة: يحتوي SwiftUI على إدارة حالة مدمجة من خلال property wrappers مثل State@ و Binding@ و ObservedObject@ و EnvironmentObject@. بينما يتطلب UIKit تتبع الحالة يدويًا وتحديثات واجهة المستخدم.
  3. عبر المنصات: يوفر SwiftUI إطارًا موحدًا لبناء واجهات المستخدم عبر جميع منصات آبل (iOS وmacOS وwatchOS وtvOS) مع الحد الأدنى من الكود الخاص بكل منصة. بينما UIKit هو في المقام الأول لـ iOS و tvOS.
  4. المعاينة المباشرة: يوفر SwiftUI معاينات في الوقت الفعلي في Xcode من خلال Canvas، مما يتيح تكرار أسرع دون الحاجة إلى بناء وتشغيل التطبيق في كل مرة.
  5. تحديثات واجهة المستخدم التلقائية: عندما تتغير الحالة في SwiftUI، يقوم الإطار تلقائيًا بتحديث واجهة المستخدم. في UIKit، يجب عليك تحديث العناصر المرئية يدويًا استجابة لتغييرات البيانات.
  6. نظام التخطيط: يستخدم SwiftUI نظام تخطيط حديث مع المكدسات (VStack و HStack و ZStack) والمحاذاة. بينما يستخدم UIKit نظام Auto Layout مع القيود.
  7. التكوين: يحتوي SwiftUI على دعم ممتاز لإنشاء مكونات واجهة مستخدم قابلة لإعادة الاستخدام وقابلة للتركيب.

لاحظ في المثال المقدم كيف أن كتابة نفس الوظيفة (نص مع زر تبديل يغير لون النص) تتطلب كودًا أقل بكثير في SwiftUI مقارنة بـ UIKit. في SwiftUI، يتم تحديث واجهة المستخدم تلقائيًا عندما تتغير الحالة (isToggled)، بينما في UIKit يجب عليك كتابة كود خاص لإدارة التغييرات والتحديثات.

بشكل عام، يمثل SwiftUI تطورًا كبيرًا في تطوير تطبيقات آبل، حيث يسمح بكتابة كود أقل وأكثر وضوحًا، مع تقليل الأخطاء المحتملة وتحسين تجربة المطور.
