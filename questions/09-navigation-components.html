What is the difference between NavigationStack, NavigationView, NavigationLink?

Answer in English

NavigationView (Deprecated in iOS 16):

The old way to create navigation in SwiftUI. It had limitations and inconsistent behavior across platforms.

NavigationView {
    List(items) { item in
        NavigationLink(destination: DetailView(item: item)) {
            Text(item.name)
        }
    }
    .navigationTitle("Items")
}

NavigationStack (iOS 16+):

The modern replacement for NavigationView with better programmatic control and type-safe navigation.

NavigationStack {
    List(items) { item in
        NavigationLink(value: item) {
            Text(item.name)
        }
    }
    .navigationDestination(for: Item.self) { item in
        DetailView(item: item)
    }
    .navigationTitle("Items")
}

NavigationLink:

A view that triggers navigation to another view. Works with both NavigationView and NavigationStack.

// Old way (with NavigationView)
NavigationLink(destination: DetailView()) {
    Text("Go to Detail")
}

// New way (with NavigationStack)
NavigationLink(value: "detail") {
    Text("Go to Detail")
}

Key Improvements in NavigationStack:

Feature NavigationView NavigationStack
Programmatic Navigation Limited, hacky solutions Built-in with NavigationPath
Type Safety AnyView destinations Type-safe with value-based navigation
Deep Linking Difficult to implement Easy with path manipulation
Performance All destinations created upfront Lazy destination creation

Programmatic Navigation with NavigationStack:

struct ContentView: View {
    @State private var path = NavigationPath()
    
    var body: some View {
        NavigationStack(path: $path) {
            VStack {
                Button("Go to Settings") {
                    path.append("settings")
                }
                Button("Go to Profile") {
                    path.append("profile")
                }
            }
            .navigationDestination(for: String.self) { value in
                switch value {
                case "settings":
                    SettingsView()
                case "profile":
                    ProfileView()
                default:
                    Text("Unknown destination")
                }
            }
        }
    }
}

Migration Guidelines:

  • Replace NavigationView with NavigationStack for iOS 16+
  • Use value-based NavigationLinks instead of destination-based
  • Implement navigationDestination for each data type
  • Use NavigationPath for programmatic navigation

الشرح بالعربي

NavigationView (مهجور من iOS 16):

دا الطريقة القديمة للتنقل في SwiftUI. كان فيه مشاكل وسلوك غير متسق بين الأجهزة.

NavigationStack (iOS 16+):

دا البديل الجديد والمحسن، بيوفر تحكم برمجي أفضل وتنقل آمن من ناحية الأنواع.

NavigationLink (رابط التنقل):

دا الـ view اللي بيخليك تنتقل من صفحة لصفحة. بيشتغل مع الاتنين، بس بطرق مختلفة.

التحسينات الجديدة في NavigationStack:

  • التحكم البرمجي: دلوقتي ممكن تتحكم في التنقل من الكود بسهولة
  • الأمان: مافيش أخطاء في أنواع البيانات
  • الأداء: الصفحات بتتعمل بس لما تحتاجها
  • Deep Linking: سهل جداً تعمل روابط عميقة

كيفية الهجرة:

  1. استبدل NavigationView بـ NavigationStack
  2. استخدم NavigationLink مع values بدل destinations
  3. استخدم navigationDestination لكل نوع بيانات
  4. استخدم NavigationPath للتحكم البرمجي

مثال عملي للتحكم البرمجي:

@State private var path = NavigationPath()

// للانتقال لصفحة معينة
path.append("settings")

// للرجوع للخلف
path.removeLast()

// للرجوع للصفحة الرئيسية
path = NavigationPath()

نصيحة: لو تطبيقك يدعم iOS 16+، استخدم NavigationStack. لو محتاج دعم إصدارات أقدم، استخدم NavigationView مع if #available.
