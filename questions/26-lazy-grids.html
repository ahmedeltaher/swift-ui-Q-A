What is LazyVGrid and LazyHGrid, and how are they different from List?

Answer in English

LazyVGrid and LazyHGrid Overview:

LazyVGrid and LazyHGrid are container views that arrange child views in a grid with lazy loading. They only create views that are visible on screen, making them memory-efficient for large datasets.

LazyVGrid (Vertical Grid):

Arranges items in columns, scrolling vertically:

struct PhotoGrid: View {
    let photos = Array(1...100).map { "photo\($0)" }
    
    let columns = [
        GridItem(.flexible()),
        GridItem(.flexible()),
        GridItem(.flexible())
    ]
    
    var body: some View {
        ScrollView {
            LazyVGrid(columns: columns, spacing: 20) {
                ForEach(photos, id: \.self) { photo in
                    AsyncImage(url: URL(string: photo))
                        .frame(height: 100)
                        .background(Color.gray)
                        .cornerRadius(8)
                }
            }
            .padding()
        }
    }
}
LazyVGrid Layout:
┌─────┬─────┬─────┐
│ 1 │ 2 │ 3 │
├─────┼─────┼─────┤
│ 4 │ 5 │ 6 │
├─────┼─────┼─────┤
│ 7 │ 8 │ 9 │
└─────┴─────┴─────┘
↓ Scrolls vertically

LazyHGrid (Horizontal Grid):

Arranges items in rows, scrolling horizontally:

struct CategoryGrid: View {
    let categories = Array(1...50).map { "Category \($0)" }
    
    let rows = [
        GridItem(.fixed(60)),
        GridItem(.fixed(60)),
        GridItem(.fixed(60))
    ]
    
    var body: some View {
        ScrollView(.horizontal) {
            LazyHGrid(rows: rows, spacing: 20) {
                ForEach(categories, id: \.self) { category in
                    Text(category)
                        .frame(width: 120, height: 50)
                        .background(Color.blue)
                        .cornerRadius(8)
                }
            }
            .padding()
        }
    }
}
LazyHGrid Layout:
┌─────┬─────┬─────┬─────┐
│ 1 │ 4 │ 7 │ 10 │ →
├─────┼─────┼─────┼─────┤
│ 2 │ 5 │ 8 │ 11 │ →
├─────┼─────┼─────┼─────┤
│ 3 │ 6 │ 9 │ 12 │ →
└─────┴─────┴─────┴─────┘
Scrolls horizontally →

GridItem Types:

// Fixed size
GridItem(.fixed(100))           // Exactly 100 points

// Flexible size
GridItem(.flexible())           // Takes available space
GridItem(.flexible(minimum: 50, maximum: 200))

// Adaptive size
GridItem(.adaptive(minimum: 80)) // Fits as many as possible with min size

Comparison with List:

Feature List LazyVGrid/LazyHGrid
Layout Single column, vertical Multiple columns/rows, customizable
Built-in Features Selection, swipe actions, refresh No built-in interactions
Appearance Table-like with separators Grid layout, no separators
Lazy Loading Yes Yes
Customization Limited to list style Highly customizable grid
Use Cases Settings, contacts, feeds Photo galleries, product grids

When to Use Each:

Use List when:
  • You need a traditional table view
  • You want built-in selection and swipe actions
  • You're displaying text-heavy content
  • You need sections with headers/footers
Use LazyVGrid when:
  • You need a photo/image gallery
  • You want multiple columns
  • You're creating a card-based layout
  • You need custom grid arrangements
Use LazyHGrid when:
  • You need horizontal scrolling content
  • You're creating category browsing
  • You want horizontal card layouts
  • You need multiple rows with horizontal flow

Performance Considerations:

// ✅ Good - Lazy loading with proper IDs
LazyVGrid(columns: columns) {
    ForEach(items, id: \.id) { item in
        ItemView(item: item)
    }
}

// ❌ Avoid - Creating all views at once
VStack {
    ForEach(items) { item in
        ItemView(item: item)  // Creates all views immediately
    }
}

Advanced Grid Features:

// Adaptive columns that adjust to screen size
let adaptiveColumns = [GridItem(.adaptive(minimum: 150))]

LazyVGrid(columns: adaptiveColumns, spacing: 20) {
    ForEach(items) { item in
        ItemCard(item: item)
            .aspectRatio(1, contentMode: .fit)
    }
}

// Mixed column sizes
let mixedColumns = [
    GridItem(.fixed(100)),
    GridItem(.flexible()),
    GridItem(.fixed(50))
]

الشرح بالعربي

إيه هي LazyVGrid و LazyHGrid؟

دول containers في SwiftUI بيرتبوا الـ views في شكل grid (شبكة) مع lazy loading - يعني بيعملوا الـ views بس لما تظهر على الشاشة.

LazyVGrid (الشبكة العمودية):

بيرتب العناصر في أعمدة، والتمرير بيكون عمودي (من فوق لتحت):

شكل LazyVGrid:
┌─────┬─────┬─────┐
│ 1 │ 2 │ 3 │
├─────┼─────┼─────┤
│ 4 │ 5 │ 6 │
├─────┼─────┼─────┤
│ 7 │ 8 │ 9 │
└─────┴─────┴─────┘
↓ التمرير عمودي

LazyHGrid (الشبكة الأفقية):

بيرتب العناصر في صفوف، والتمرير بيكون أفقي (من اليمين للشمال):

شكل LazyHGrid:
┌─────┬─────┬─────┬─────┐
│ 1 │ 4 │ 7 │ 10 │ ←
├─────┼─────┼─────┼─────┤
│ 2 │ 5 │ 8 │ 11 │ ←
├─────┼─────┼─────┼─────┤
│ 3 │ 6 │ 9 │ 12 │ ←
└─────┴─────┴─────┴─────┘
التمرير أفقي ←

أنواع GridItem:

  • Fixed: حجم ثابت
  • Flexible: حجم مرن حسب المساحة المتاحة
  • Adaptive: يحط أكبر عدد ممكن من العناصر

الفرق بين Grid و List:

List (القائمة):
  • عمود واحد بس
  • شكل تقليدي زي الجداول
  • فيه مميزات جاهزة زي الاختيار والسحب
  • مناسب للنصوص والقوائم
LazyVGrid/LazyHGrid (الشبكة):
  • أعمدة أو صفوف متعددة
  • تحكم كامل في الشكل
  • مناسب للصور والكروت
  • مرونة أكبر في التصميم

متى تستخدم إيه:

استخدم List لما:

  • عايز قائمة تقليدية زي الإعدادات
  • محتاج مميزات زي الاختيار والسحب
  • المحتوى نصوص أساساً

استخدم LazyVGrid لما:

  • عايز معرض صور
  • محتاج أعمدة متعددة
  • بتعمل layout للكروت

استخدم LazyHGrid لما:

  • عايز التمرير يكون أفقي
  • بتعمل categories أو فئات
  • محتاج صفوف متعددة أفقية

مثال عملي:

// شبكة صور بثلاث أعمدة
let columns = [
    GridItem(.flexible()),
    GridItem(.flexible()),
    GridItem(.flexible())
]

LazyVGrid(columns: columns) {
    ForEach(photos) { photo in
        AsyncImage(url: photo.url)
            .aspectRatio(1, contentMode: .fill)
    }
}

مميزات الـ Lazy Loading:

  • بيوفر في الذاكرة
  • أداء أفضل مع البيانات الكبيرة
  • تمرير سلس
  • بيعمل الـ views بس لما تحتاجها

خلاصة: الـ Grids مثالية للمحتوى المرئي زي الصور والكروت، بينما الـ List أفضل للقوائم التقليدية!
