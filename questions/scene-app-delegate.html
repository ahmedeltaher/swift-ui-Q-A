What is the role of SceneDelegate and AppDelegate in SwiftUI apps?

Answer in English

AppDelegate:

AppDelegate manages the application's lifecycle and handles app-level events. In SwiftUI apps, it's still used for:

  • Application launch setup
  • Push notification registration
  • Background tasks
  • Third-party SDK initialization
@main
struct MyApp: App {
    @UIApplicationDelegateAdaptor(AppDelegate.self) var appDelegate
    
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}

class AppDelegate: NSObject, UIApplicationDelegate {
    func application(_ application: UIApplication, 
                   didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]? = nil) -> Bool {
        // Setup code
        return true
    }
}

SceneDelegate:

SceneDelegate manages the scene lifecycle in multi-window apps (mainly iPad). It handles:

  • Scene creation and configuration
  • Window management
  • Scene transitions (background/foreground)
class SceneDelegate: UIResponder, UIWindowSceneDelegate {
    var window: UIWindow?

    func scene(_ scene: UIScene, willConnectTo session: UISceneSession, 
              options connectionOptions: UIScene.ConnectionOptions) {
        // Scene setup
        if let windowScene = scene as? UIWindowScene {
            let window = UIWindow(windowScene: windowScene)
            window.rootViewController = UIHostingController(rootView: ContentView())
            self.window = window
            window.makeKeyAndVisible()
        }
    }
}

SwiftUI App Protocol (iOS 14+):

Modern SwiftUI apps can use the App protocol instead of AppDelegate/SceneDelegate:

@main
struct MySwiftUIApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
        .onChange(of: scenePhase) { phase in
            switch phase {
            case .active:
                // App became active
            case .inactive:
                // App became inactive
            case .background:
                // App went to background
            @unknown default:
                break
            }
        }
    }
}

الشرح بالعربي

AppDelegate (مفوض التطبيق):

دا المسؤول عن إدارة التطبيق كله من البداية للنهاية. حتى في SwiftUI لسه مهم عشان:

  • إعداد التطبيق عند البدء
  • تسجيل الـ Push Notifications
  • المهام في الخلفية
  • إعداد المكتبات الخارجية

SceneDelegate (مفوض المشهد):

دا مسؤول عن إدارة النوافذ المختلفة، خاصة في الـ iPad اللي ممكن يفتح أكتر من نافذة لنفس التطبيق.

التطور في SwiftUI:

من iOS 14، SwiftUI جابت حاجة اسمها App Protocol اللي بيخليك تستغنى عن AppDelegate و SceneDelegate في معظم الحالات.

متى تستخدم إيه:

  • App Protocol: للتطبيقات البسيطة والحديثة
  • AppDelegate: لما تحتاج تتعامل مع مكتبات قديمة أو push notifications
  • SceneDelegate: للتطبيقات اللي تدعم multi-window

نصيحة: ابدأ بـ App Protocol، ولو احتجت حاجات متقدمة، ضيف AppDelegate باستخدام @UIApplicationDelegateAdaptor.
